Code review (đánh giá mã nguồn)

Ba mô-đun: (1) review một PR — bấm dòng nghi vấn để hiện vấn đề; (2) phân loại comment blocking / nit / praise; (3) PR nhỏ hay to — kéo slider xem chi phí review & tỉ lệ lọt lỗi.

1 · Review một PR — bấm dòng nghi vấn

Đoạn code có vài vấn đề. Dòng có chấm vàng là chỗ đáng ngờ — bấm vào để xem reviewer sẽ comment gì. Tìm hết các vấn đề!

2 · Phân loại comment review

Mỗi comment thuộc loại nào? Blocking = phải sửa (request changes); Nit/Suggestion = gợi ý không chặn; Praise = khen. Bấm chọn để kiểm tra.

3 · PR nhỏ hay to? — kéo slider số dòng

Kéo để đổi kích thước PR. Xem thời gian review ước tính, hành vi thực tế của reviewer, và tỉ lệ lỗi bị bỏ sót thay đổi ra sao.
