# Nhóm 1 — Cơ bản — Cấu trúc dữ liệu cơ bản

Những cấu trúc dữ liệu gặp hàng ngày trong code và phần lớn bài phỏng vấn. Đây là nền tảng cho mọi cấu trúc phức tạp hơn ở các nhóm sau.

## Mục tiêu

- Đọc và viết Big-O của thuật toán một cách tự tin.
- Chọn được giữa Array vs Linked List, Stack vs Queue tùy bài toán.
- Hiểu vì sao Hash Table cho trung bình O(1) — và lúc nào nó tệ.

## Lộ trình 6 bài

| Bài | Chủ đề | Khái niệm chính |
|-----|--------|-----------------|
| [Lesson 01](./lesson-01-introduction/) | Giới thiệu | Khái niệm, phân loại, Big-O chi tiết |
| [Lesson 02](./lesson-02-array/) | Array | Mảng tĩnh / động, truy cập theo chỉ số, amortized O(1) |
| [Lesson 03](./lesson-03-linked-list/) | Linked List | Singly, doubly, circular; reverse, Floyd cycle |
| [Lesson 04](./lesson-04-stack/) | Stack | LIFO, kiểm tra ngoặc, postfix, shunting-yard |
| [Lesson 05](./lesson-05-queue/) | Queue | FIFO, circular queue, deque, priority queue |
| [Lesson 06](./lesson-06-hash-table/) | Hash Table | Hash function, collision, load factor, polynomial hash |

Mỗi bài có:
- **README.md**: lý thuyết + ví dụ số cụ thể + bài tập + **lời giải chi tiết**.
- **solutions.go**: code Go biên dịch được (`go run solutions.go`).
- **visualization.html**: trang tương tác, mở trực tiếp trong trình duyệt là chạy. Có nút **📖 Đọc README** để xem lý thuyết song song.
