Lesson 07 — Công cụ chẩn đoán mạng

Bài này mô phỏng 3 loại công cụ chính: Terminal giả lập (ping, traceroute, dig, netstat, curl), Wireshark mini (phân tích gói bắt tay TCP + HTTP), và Cây quyết định chẩn đoán "web không vào được". Nhấn vào lệnh hoặc nhập trực tiếp để tương tác.

1. Terminal giả lập — ping, traceroute, dig, netstat, curl

Chọn lệnh có sẵn hoặc nhập vào terminal. Lệnh hỗ trợ: ping, traceroute, dig, nslookup, netstat, ss, curl, help, clear.

user@linux:~$
Chào mừng đến Terminal mô phỏng Networking Tools.
Nhập lệnh hoặc nhấn nút bên trên. Gõ help để xem danh sách lệnh.
user@linux:~$ 

2. Wireshark mini — Phân tích phiên TCP + HTTP

Bảng dưới mô phỏng output Wireshark khi bắt một phiên HTTP hoàn chỉnh (3-way handshake + GET/200 + 4-way close). Nhấn vào một hàng để xem chi tiết tầng. Dùng bộ lọc để lọc gói.
