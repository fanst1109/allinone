Ba module tương tác: (1) so sánh dòng message của Long polling / SSE / WebSocket; (2) handshake nâng cấp HTTP → WS từng bước; (3) chat broadcast nhiều client qua hub. Mở README (nút 📖 góc phải) để đọc lý thuyết đầy đủ.

1. So sánh dòng message — Long polling / SSE / WebSocket

Long polling

Mỗi event = 1 HTTP request mới

SSE

1 connection giữ mãi, server đẩy 1 chiều

WebSocket

Full-duplex, hai chiều đồng thời
ClientServer
Quan sát: Long polling tốn 1 request setup + headers cho mỗi event (gói xanh dương đi lên trước mỗi gói xanh lá). SSE chỉ 1 request đầu, sau đó server đẩy thẳng. WebSocket cả 2 chiều cùng bay — client gửi typing indicator trong khi server đẩy message.

3. Chat broadcast — hub-and-spoke nhiều client

Hub
sở hữu map
register · unregister
broadcast
Chỉ goroutine Hub đụng vào map clients → không cần mutex. Khi 1 client gửi, message đi vào broadcast channel; Hub phát vào send channel (buffered) của từng client. Nếu buffer client đầy (client chậm) → Hub drop client đó thay vì block cả hệ thống. Bấm 🐌 để xem Carol bị drop.
