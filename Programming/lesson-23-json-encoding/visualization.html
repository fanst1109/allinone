Lesson 23 — JSON Encoding với encoding/json

3 module dưới đây minh hoạ (1) tag struct ảnh hưởng output JSON ra sao, (2) quá trình Unmarshal match field giữa JSON và struct, (3) sự khác biệt RAM giữa ReadAll + Unmarshal (load-all) và NewDecoder + Decode (stream) trên file 100MB.

3. Stream vs Load-all — RAM khi parse file 100MB

Giả lập đọc file NDJSON 100MB (1 triệu user JSON). So sánh 2 cách: Load-all (io.ReadAll + json.Unmarshal) — toàn bộ file vào RAM trước khi parse; vs Stream (json.NewDecoder.Decode) — đọc nhỏ giọt, chỉ giữ 1 object tại 1 thời điểm.

Trục RAM scale theo file size. Bạn sẽ thấy Load-all tăng tuyến tính theo kích thước file (RAM spike = full file size), trong khi Stream giữ phẳng ở mức ~vài chục KB bất kể file lớn cỡ nào.
Tại sao quan trọng? Server có 1GB RAM, nhận request upload JSON 600MB. Load-all → out-of-memory crash. Stream → chạy thoải mái. Đây là lý do mọi HTTP framework dạy json.NewDecoder(r.Body).Decode(&req), không phải io.ReadAll(r.Body) trước.
