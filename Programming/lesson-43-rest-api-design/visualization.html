L43 — REST API Design

Ba module tương tác để nắm vững thiết kế REST: (1) chọn action → suy ra HTTP method + URL + status; (2) so sánh 3 kiểu pagination (offset / page / cursor) bằng animation; (3) quiz status code với 15 tình huống thực tế.

1. HTTP Method Picker — action → method + URL + status

URL mô tả resource (danh từ), HTTP method mô tả action. Chọn một mục đích bên dưới để xem cách map đúng chuẩn REST, kèm cờ safe / idempotent.

Chọn một action ở trên để xem mapping REST chuẩn.

2. Pagination Comparison — offset vs page vs cursor

Bảng có 20 record (id 1..20). Chọn kiểu pagination rồi bấm Trang sau để xem cách mỗi chiến lược lấy dữ liệu — và vì sao cursor ổn định khi insert/delete.

3. Status Code Picker — 15 tình huống

Chọn một tình huống để xem status code đúng và lý do. Bao trùm 2xx / 3xx / 4xx / 5xx.

Chọn một tình huống ở trên để xem status đúng.
