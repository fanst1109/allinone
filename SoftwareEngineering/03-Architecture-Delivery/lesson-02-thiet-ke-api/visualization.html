Thiết kế API — REST, gRPC, GraphQL

Ba mô-đun: (1) chấm điểm endpoint REST theo best practice, (2) so sánh payload REST (lấy thừa) vs GraphQL (chọn field), (3) quiz chọn HTTP status code.

1 · REST endpoint — chấm điểm theo best practice

Chọn HTTP verb + gõ URL, rồi bấm "Chấm điểm". Hệ thống kiểm tra các quy tắc đặt URL (danh từ số nhiều, không động từ trong URL, không dùng GET để đổi dữ liệu...).

Thử nhanh: GET /users/42 GET /getUser?id=42 POST /createUser GET /deleteUser?id=42 POST /users/42/orders

    2 · Over-fetching: REST trả hết vs GraphQL chọn field

    Màn hình của bạn chỉ cần vài field. REST (1 endpoint) trả về cả 6 field; GraphQL trả đúng field bạn chọn. Tích/bỏ field để xem chênh lệch kích thước payload.
