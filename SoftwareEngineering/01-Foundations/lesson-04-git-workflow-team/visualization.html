Git workflow cho nhóm

Ba mô-đun: (1) dựng commit graph bằng nút bấm, (2) GitFlow vs Trunk-Based bật/tắt, (3) giải merge conflict.

1 · Dựng commit graph — tạo branch / commit / merge

Bấm các nút để thấy đồ thị nhánh hình thành. main là dòng dưới; tạo branch để rẽ nhánh, commit để thêm node, merge để gộp về main.

main bắt đầu với 1 commit. Tạo một feature branch rồi commit lên nó, sau đó merge về main.

2 · GitFlow vs Trunk-Based

Bấm để chuyển. Diagram và bảng so sánh đổi theo chiến lược.

🌿 GitFlow 🌳 Trunk-Based

3 · Giải merge conflict

Git dừng lại vì hai người sửa cùng dòng. Đọc cả hai phía rồi chọn cách hợp nhất đúng nghiệp vụ. Bối cảnh: phía của bạn đổi thuế xuống 8%; phía nhánh kia thêm làm tròn. Cả hai ý đều cần giữ.

Chọn một phương án hợp nhất.
