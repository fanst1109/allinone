L77 — CI/CD Pipeline

Từ commit tới production: chạy thử pipeline stages (fail-fast), so sánh 4 deploy strategy, và mô phỏng canary rollout tự promote/rollback theo metric.

1. Pipeline stages — fail-fast

Mỗi commit chạy qua chuỗi stage. Stage trước fail → dừng luôn, các stage sau bị skip (tiết kiệm runner). Chọn stage để ép fail rồi bấm "Chạy pipeline" xem nó dừng ở đâu.

2. Deploy strategies

Cùng thay 4 instance từ v1 (xanh dương) sang v2 (xanh lá). Mỗi strategy đánh đổi khác nhau về downtime, tài nguyên, tốc độ rollback.

3. Canary rollout — metric-based promote/rollback

Canary nhận % traffic tăng dần (5% → 25% → 50% → 100%). Mỗi bước đo error rate giả lập của canary; ≤ ngưỡng → promote (tăng %), > ngưỡng → rollback ngay. Kéo "Lỗi canary mô phỏng" để xem hệ thống tự quyết định.
